# Task 002 — GitHub Hardening

## Objective

Validate GitHub workflow files, templates, and documentation. Do not introduce app logic.

## Scope

- Ensure `.github/workflows/*` are syntactically valid YAML.
- Ensure PR and issue templates are present.
- Ensure `.gitignore` excludes private Obsidian data.
- Ensure docs explain branch protection and secret protection.

## Acceptance criteria

- GitHub Actions files are valid YAML.
- No private data is tracked.
- README links to GitHub workflow docs.
