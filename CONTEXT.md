# Project Context

## Project name
Affiliate Product Intelligence OS

## Mission
Build a local-first AI operating system that helps identify high-opportunity affiliate products before creating content.

The system must prioritize product discovery, demand intelligence, trend velocity, marketplace ranking, commission economics, content fit, competition gap, and compliance risk.

## Phase 1 architecture

- Backend/project memory: Obsidian Vault only.
- Storage format: Markdown notes with YAML frontmatter.
- AI orchestration: Hermes Agent.
- Implementation: Codex CLI.
- Agent coordination: tmux warroom.
- Source control and engineering workflow: GitHub.
- Vendor skill library: Affitor affiliate-skills.

## Non-goals in Phase 1

- No production cloud deployment.
- No database.
- No OAuth/RBAC.
- No autopublish.
- No scraping that may violate marketplace terms.
- No real credential storage in GitHub.
