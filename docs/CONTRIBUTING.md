# Contributing

## Development rules

- Read `AGENTS.md` first.
- Keep changes small.
- Add tests for logic.
- Do not commit secrets.
- Do not commit private vault data.
- Do not add a database in Phase 1.

## Before PR

- Run available tests.
- Check `git diff` for secrets.
- Confirm `.env` is not staged.
- Confirm private `vault/*` notes are not staged.
