# Branching Strategy

## Main branches

- `main`: stable branch
- `feature/<short-name>`: feature work
- `fix/<short-name>`: fixes
- `docs/<short-name>`: documentation
- `chore/<short-name>`: maintenance

## Rules

- Codex must work on non-main branches.
- Every implementation must be tied to a task file or GitHub issue.
- Merge through Pull Request.
