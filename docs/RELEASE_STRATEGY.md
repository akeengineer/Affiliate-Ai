# Release Strategy

## Versioning

Use simple semantic versioning after MVP:

- `v0.1.0`: scoring + Obsidian workflow
- `v0.2.0`: weekly report + tmux warroom
- `v0.3.0`: content draft + compliance
- `v0.4.0`: dashboard

## Release checklist

- tests pass
- docs updated
- no secrets in git
- sample data only
- changelog updated
